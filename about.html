🏕️ 关于本站

荒野之息攻略站是一个由玩家社区驱动的《塞尔达传说：旷野之息》攻略站点。我们致力于为所有玩家提供最全面、最准确、最易于阅读的游戏攻略内容。

🎯 我们的使命：让每一位踏入海拉鲁的勇者，无论语言障碍、无论游戏经验，都能在这片广袤的大陆上找到属于自己的冒险之路。

这个站点是为谁准备的？

无论你是刚刚从复苏神庙醒来的新手勇者，还是已经收集了全部900个呀哈哈的老练冒险家——这里都有属于你的内容。

  • 新手玩家——从初始台地到海拉鲁城堡，手把手带你走完主线
  • 收集党——120座神庙、全部装备、隐藏要素，一个不落
  • 战斗爱好者——人马速杀、盾反教学、BOSS无伤攻略
  • 料理大师——全料理配方与最佳属性搭配
💡 社区驱动：本站内容完全开源，所有攻略均由社区玩家共同编写和维护。没有广告、没有付费墙——因为我们相信，攻略知识应该对所有人免费开放。

目前我们已收录了120座神庙的完整解法、四大神兽的主线攻略、全装备图鉴、料理配方大全以及大量隐藏要素和彩蛋。但我们知道，海拉鲁的秘密远不止于此——还有更多内容等待我们去发现和记录。

✍️ 贡献攻略

荒野之息攻略站是一个开源项目，托管在 GitHub 上。每一篇攻略、每一处修正、每一张配图，都来自像你一样热爱这款游戏的玩家。我们热忱欢迎你的加入！

📂 GitHub 仓库：https://github.com/Brunooooooooooooooo/zelda-breath-of-the-wild-guide

如何贡献？

  1. Fork 仓库——点击 GitHub 页面右上角的 Fork 按钮，将项目复制到你的账号下
  2. 创建分支——为你的贡献创建一个新的分支，例如 add-guardian-guide
  3. 编写内容——在本地编辑器中完成你的攻略内容。所有页面使用标准的 HTML5 格式，内容区域使用 <section class="guide-section"> 包裹
  4. 提交 Pull Request——推送到你的分支后，在 GitHub 上发起 Pull Request，我们会尽快审核

内容指南

💡 写作原则：准确 > 全面 > 美观。确保信息无误是第一位的，格式可以后续优化。
  • 准确性——所有攻略信息需要经过实际游戏验证，请勿提交未经测试的内容
  • 多语言优先——本站支持五语言切换，撰写攻略时请尽量使用清晰简洁的表达以便翻译
  • 清晰结构——使用合理的标题层级（h2/h3/h4），善用表格和列表提升可读性
  • 尊重版权——配图请使用你自己的游戏截图，或确保图片来源符合使用许可
  • 友善协作——Code Review 过程中我们可能会提出修改建议，请保持开放心态

欢迎哪些类型的贡献？

  • 🆕 新增攻略——尚未覆盖的游戏内容、DLC攻略、进阶技巧
  • 🔧 内容修正——发现错误信息？提交修正让攻略更准确
  • 🎨 排版优化——改善页面结构、提升阅读体验
  • 📸 截图补充——为攻略添加关键位置的截图和标注
  • 🌐 翻译校对——统一术语、优化中文表达
  • 🐛 技术改进——网站功能、样式、性能优化

📬 反馈建议

你发现了攻略中的错误？有想要看到的攻略内容？对网站有什么改进建议？我们非常乐意听到你的声音！

反馈方式

🐛 提交 Issue：在 GitHub Issues 中创建新 Issue，这是最推荐的方式，方便我们跟踪和处理。

提交反馈时，请尽量包含：

  • 具体位置——出错的页面链接或攻略章节名称
  • 问题描述——清晰说明你发现的问题或期望的改进
  • 复现步骤（如果是错误）——如何触发该问题
  • 建议方案（可选但欢迎）——如果你有解决方案，欢迎一并提出
  • 游戏版本——你使用的游戏版本（普通版/大师模式/DLC等），某些信息可能因版本而异
💡 小建议：在提交 Issue 之前，可以先搜索一下是否已有类似的反馈。在已有 Issue 下补充信息，比创建重复 Issue 更有效率。

我们会定期查看所有反馈，通常会在1-2周内给出回复。对于紧急错误（如严重误导性信息），会优先处理。
